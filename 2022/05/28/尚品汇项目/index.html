尚品汇项目记录

一、使用Vue-cli初始化项目

1.创建脚手架

在项目文件夹cmd输入vue create app(项目名称)

文件夹 功能
node_modules文件夹 存放项目依赖文件
public文件夹 放置静态资源(图片)，注意：当中的资源，webpack在进行打包时，会原封不动打包到dist文件夹
–src文件夹(源代码文件夹)：assets文件夹 一般放置多个组件共用的静态资源，在webpack打包时，会将其当作一个模块，打包到JS文件中
–src文件夹(源代码文件夹)：components文件夹 一般放置非路由组件（全局组件）
–src文件夹(源代码文件夹)：App.vue 唯一的根组件
–src文件夹(源代码文件夹)：main.js 程序入口文件，最先执行的文件
babel.config.js 语法翻译，提高兼容性
package.json 项目的信息记录
package-lock.json 缓存性文件，记录依赖文件下载地址等

2.项目的其他配置

​ 1.浏览器自动打开：package.json中的scripts中的serve配置项改为vue-cli-service serve --open

​ 2.关闭eslint代码校验：在vue.config.js中，lintOnSave: false

eslint：声明变量但未使用，就会报错

​ 3.给 src 文件夹配置别名@，方便编程时的文件引用

二、项目路由的分析

路由就是键值对，key为地址栏中的路径，value为相应的路由组件

​ 1.路由组件：Home首页路由组件、search路由组件、Login登录路由、Register注册路由

​ 2.非路由组件：Header、Footer（在登录、注册页面没有）

三、完成非路由组件Header和Footer业务

开发项目：1.书写静态页面（html+css） 2.拆分组件 3.获取服务器的数据动态展示 4.完成相应业务逻辑

注意

​ 1.创建组件时，组件结构 + 组件样式 + 图片资源

​ 2.项目采用的是less样式，浏览器不能识别less样式，需要通过less、less-loader进行处理，转为css样式

​ 3.如果希望组件识别less样式，需要在style标签上添加lang="less"

1.使用组件的步骤（非路由组件）

  1. 创建或定义组件
  2. 引入组件
  3. 注册组件
  4. 用标签的形式使用组件

四、路由组件的搭建

组件应当有四个：Home、Search、Login、Register

1.建立pages文件夹

–components文件夹放非路由组件（共用全局组件）

–pages文件夹放路由组件

2.配置路由

创建router文件夹，新建router.js

3.在main.js中引入路由

4.在根组件书写路由组件出口

在App.vue中使用<router-view></router-view>将路由组件引出

5.总结

5.1 路由组件与非路由组件的区别

1.路由组件一般放置在pages文件夹，非路由组件一般放置在components文件夹

2.路由组件一般需要在router文件夹中进行注册（使用的即为组件的名字），非路由组件在使用的时候，一般都是以标签的形式使用

3.注册完路由，不管是路由组件还是非路由组件，身上都有$router，$router属性

$route：一般获取路由信息【路径、query、params等】

$router：一般进行编程式导航进行路由跳转【push|replace】 两者区别：能否记住历史纪录，push追加历史记录，replace替换当前记录

5.2 路由的跳转

路由的跳转有两种形式：

1.声明式导航router-link

2.编程式导航push|replace：声明式导航能做的，编程式导航都能做，除此之外，用在处理js逻辑之后进行的页面跳转
